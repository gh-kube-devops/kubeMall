## 🧱 整体架构

```
Frontend (Vue3)
        ↓
     Gateway
        ↓
 ┌──────────────┬──────────────┬──────────────┐
 ↓              ↓              ↓
User        Product        Order
Service     Service        Service
 ↓              ↓              ↓
MySQL        MySQL          MySQL
        ↓
      Redis
```

## 🔥 最小可运行功能

### 用户模块

* 注册
* 登录

### 商品模块

* 商品列表
* 商品详情

### 订单模块

* 下单
* 查询订单

👉 做完这些，系统就“跑通了”

---

## 🔗 接口调用关系

```
前端 → Gateway → user-service
前端 → Gateway → product-service
前端 → Gateway → order-service
                    ↓
            调用 product-service
            调用 user-service
```

## 🐳 Docker 容器化

```dockerfile
FROM openjdk:17
COPY target/app.jar app.jar
ENTRYPOINT ["java","-jar","/app.jar"]
```

## ☸️ Kubernetes 部署

每个服务：

* Deployment
* Service
* ConfigMap（可选）

---

## 🌍 Ingress 入口

```
/api/user    → user-service
/api/product → product-service
/api/order   → order-service
```

## 🧠 可扩展与裁剪

### ❌ 初期可以不做

* RabbitMQ
* Kafka
* Nacos
* 分布式事务

### ✅ 后期可加入

* Redis
* JWT 登录
* 限流
* 日志系统

### 🚀 可扩展服务

* payment-service
* stock-service
* 秒杀系统

---

## 🎯 推荐实施步骤

### 🥇 第一步：单体应用（必须先做）

👉 所有功能写在一个 Spring Boot

---

### 🥈 第二步：拆微服务

* user-service
* product-service
* order-service
* gateway

---

### 🥉 第三步：Docker 化

* 每个服务构建镜像

---

### 🏆 第四步：Kubernetes 部署

* Deployment
* Service
* Ingress

---

## ⭐ 总结

```
单体 → 微服务 → Docker → Kubernetes
```

👉 不要跳步骤，否则项目很容易崩
