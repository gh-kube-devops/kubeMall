# KubeMall 项目开发分工清单（DevOps转型版）

## 一、必须自己掌控（核心能力区）

### 1. 系统架构设计（必须自己完成）

* 技术选型（Vue3 / Spring Boot / PostgreSQL / Redis / K8s）
* 微服务拆分（用户服务 / 商品服务 / 订单服务）
* 系统调用链路设计（前端 → 网关 → 服务 → DB）
* 网络结构设计（Ingress / Service / Pod）

👉 要求：

* 能画架构图
* 能解释请求完整流转路径
* 能说明为什么这样设计

---

### 2. Kubernetes 核心理解（重点）

* Deployment / StatefulSet 区别
* Service（ClusterIP / NodePort / Ingress）
* ConfigMap / Secret 使用
* Pod 生命周期
* 资源限制（CPU / 内存）

👉 要求：

* 能解释每个资源作用
* 能分析服务访问路径
* 能理解调度和扩缩容逻辑

---

### 3. CI/CD 流程设计（核心）

* Git 提交触发机制
* Docker 镜像构建流程
* 镜像仓库推送
* 自动部署到 Kubernetes
* 回滚策略设计

👉 要求：

* 能画流程图（代码 → 镜像 → 部署）
* 能解释每一步作用
* 能手动模拟一次发布流程

---

### 4. 问题排查能力（DevOps核心）

* Pod 启动失败（CrashLoopBackOff）
* 服务无法访问
* 数据库连接失败
* 前端 502 / 504

👉 必会命令：
kubectl get pods
kubectl logs <pod>
kubectl describe pod <pod>
kubectl get svc
kubectl get ingress

👉 要求：

* 能独立定位问题（日志 / 网络 / 配置）

---

### 5. 核心业务逻辑（必须掌控）

* 登录 / 权限控制（JWT）
* 下单流程（订单状态流转）
* 数据一致性处理

👉 要求：

* 能讲清业务流程
* 能解释关键代码逻辑

## 四、推荐开发流程（实战顺序）

1. 本地完成后端（Spring Boot + DB）
2. 本地完成前端（Vue3）
3. Docker 化（前后端分别打镜像）
4. 编写 Kubernetes YAML
5. 部署到 K8s（Minikube / 云环境）
6. 配置 Ingress 实现外部访问
7. 接入 CI/CD（Jenkins / GitHub Actions）
8. 模拟发布 + 回滚
9. 故障演练（服务挂掉 / 网络问题）
